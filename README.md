## рҹҢҹ What is CineScope?

> *"Is this movie worth two hours of my life?"* вҖ” Everyone, before every movie.

CineScope answers that question with **real data**, not one critic's hot take.

It's a full-stack AI-powered web application that **scrapes hundreds of real user reviews** from TMDB, IMDb, and Rotten Tomatoes simultaneously, then passes each one through a **machine learning sentiment classifier** trained on 50,000 reviews. In seconds, you get a verdict backed by the collective voice of the internet.

```
You type:   "Inception"
CineScope:  Fetches 150 reviews вҶ’ Classifies each вҶ’ Returns "Positive рҹҳҠ (87.3%)"
```

No fluff. No paid subscriptions. Just raw crowd intelligence, distilled by AI.

```
       в–ј  TF-IDF vectorizer.transform()  вҶ’  sparse vector (5,000 features)
       в”Ӯ
       в–ј  LogisticRegression.predict()   вҶ’  1 (Positive) or 0 (Negative)

After all reviews:
    pos_pct = (positive_count / total) Г— 100

    pos_pct >= 60%  вҶ’  "Positive рҹҳҠ"
    pos_pct <= 40%  вҶ’  "Negative рҹҳһ"
    else            вҶ’  "Mixed рҹҳҗ"
```

### Step 5 вҖ” Rating Fallback

```
If zero reviews were found across all 3 sources:

    TMDB vote_average
        >= 7.0  вҶ’  "Positive рҹҳҠ"
        >= 4.0  вҶ’  "Mixed рҹҳҗ"
        <  4.0  вҶ’  "Negative рҹҳһ"
```

### Step 6 вҖ” Response

```
Flask renders index.html via Jinja2 with full movie_info dict
    вҶ’ Verdict banner + animated progress bar
    вҶ’ Movie card (poster, metadata)
    вҶ’ Filterable review list (All / Positive / Negative tabs)
    вҶ’ Per-review expandable cards with sentiment badge
```

## рҹӨ– Machine Learning Model

### Dataset

The model is trained on the [IMDB Dataset of 50K Movie Reviews](https://www.kaggle.com/datasets/lakshmi25npathi/imdb-dataset-of-50k-movie-reviews) вҖ” a perfectly balanced binary classification dataset.

| Property | Value |
|---|---|
| Total samples | 50,000 reviews |
| Positive reviews | 25,000 |
| Negative reviews | 25,000 |
| Train split | 40,000 (80%) |
| Test split | 10,000 (20%) |
| Vectorizer | TF-IDF, `max_features=5000` |
| Model | Logistic Regression (sklearn defaults) |
| **Test Accuracy** | **89.43%** вң… |

### Text Preprocessing Pipeline

```python
raw review
    в”Ӯ
    в–ј  text.lower()
    в”Ӯ
    в–ј  re.sub(r'<.*?>', '', text)        # strip HTML tags
    в”Ӯ
    в–ј  re.sub(r'[^a-zA-Z]', ' ', text)  # keep letters only
    в”Ӯ
    в–ј  TfidfVectorizer(max_features=5000)
    в”Ӯ
    в–ј  dense numpy array вҶ’ LogisticRegression.predict()
    в”Ӯ
    в–ј  1 = Positive  |  0 = Negative
```

### Why Logistic Regression?

Despite being a "simple" model, Logistic Regression on TF-IDF features is remarkably effective for sentiment classification:

| Advantage | Detail |
|---|---|
| вҡЎ **Fast inference** | Milliseconds per review вҖ” no GPU needed |
| рҹӘ¶ **Lightweight** | Tiny `.pkl` file, loads in seconds at startup |
| рҹҺҜ **High accuracy** | 89.43% on the standard IMDb benchmark |
| рҹ”Қ **Interpretable** | Feature weights reveal which words drive predictions |
| рҹҡҖ **Production-ready** | No runtime dependencies beyond scikit-learn |

For a web app where inference must be near-instant across 150 reviews, it's the ideal choice. Upgrading to BERT is on the roadmap вҖ” but this baseline is already excellent.

## рҹ”§ Scraper Technical Details

### IMDb Scraper

The IMDb scraper handles IMDb's frequently changing HTML layout with **3 fallback CSS selectors** in priority order:

```
1. div.ipc-html-content-inner-div     вҶҗ New layout (2023+)
2. div.text.show-more__control        вҶҗ Old layout
3. div.review-container div.content   вҶҗ Legacy layout
```

Pagination is handled via IMDb's `data-key` attribute on the load-more element. User-Agent strings are rotated across pages to reduce blocking risk.

### Rotten Tomatoes Scraper

The RT scraper first attempts to discover the movie's URL slug via RT's search endpoint, then constructs one as a fallback:

```python
"The Dark Knight"  вҶ’  "the_dark_knight"
```

Reviews are scraped from `p.audience-reviews__review` with 2 additional fallback selectors for layout changes.

### Thread Safety

All three scrapers use **per-request `requests.Session()` context managers** rather than a shared module-level session. This is critical because `review_aggregator.py` runs all three scrapers simultaneously via `ThreadPoolExecutor` вҖ” a shared session causes race conditions and intermittent connection resets.

### Deduplication Strategy

Reviews are deduplicated using a fingerprint of the first 120 characters (lowercased and stripped), stored in a Python `set`. This catches the same review appearing on multiple platforms without requiring expensive full-string comparison.

```python
from tmdb_scraper import get_tmdb_reviews
```

### вқҢ `Connection to api.themoviedb.org timed out`

**Cause:** Your network (college WiFi, office, or some mobile carriers) is blocking outbound connections to TMDB at the DNS or IP level. Python's `requests` makes direct TCP connections вҖ” your browser may still work because it uses a system proxy or different DNS resolution.

**Quick diagnosis:**

```bash
# Test TMDB вҖ” should print 200
python -c "import requests; r = requests.get('https://api.themoviedb.org', timeout=10); print(r.status_code)"

# Test general internet вҖ” should always work
python -c "import requests; r = requests.get('https://google.com', timeout=10); print(r.status_code)"
```

If Google returns `200` but TMDB times out вҖ” your network is performing DNS-level domain blocking.

#### вң… Solution 1 вҖ” Change DNS to Google DNS *(most reliable fix)*

1. Press `Win + R`, type `ncpa.cpl`, press Enter
2. Right-click your active network adapter вҶ’ **Properties**
3. Double-click **Internet Protocol Version 4 (TCP/IPv4)**
4. Select **"Use the following DNS server addresses"** and enter:

```
Preferred DNS:  8.8.8.8
Alternate DNS:  8.8.4.4
```

If `8.8.8.8` still doesn't work, try **Cloudflare DNS**:

```
Preferred DNS:  1.1.1.1
Alternate DNS:  1.0.0.1
```

**Verify the fix:**

```bash
python -c "import requests; r = requests.get('https://api.themoviedb.org/3/search/movie', params={'api_key':'your_key','query':'inception'}, timeout=10); print(r.status_code)"
# Expected: 200
```

#### вң… Solution 2 вҖ” Mobile Hotspot

Turn on hotspot on your phone, connect your laptop's WiFi to it, then run the app. If your carrier also blocks TMDB on hotspot, combine with Solution 1.

#### вң… Solution 3 вҖ” VPN

Install a free VPN, connect to any server, then run `python app.py`.

Recommended free options:
- [ProtonVPN](https://protonvpn.com) вҖ” unlimited free tier
- [Windscribe](https://windscribe.com) вҖ” 10 GB/month free

#### вң… Solution 4 вҖ” Deploy to Render

If you can't resolve local network access, deploy to Render. Render's US servers have no such blocks and the app will work perfectly in production.

### вқҢ `Invalid API key (401)`

**Cause:** Wrong API key. The TMDB settings page shows two different credentials вҖ” many users accidentally use the long Bearer token instead of the short v3 key.

**How to get the correct key:**

1. Go to [https://www.themoviedb.org/settings/api](https://www.themoviedb.org/settings/api) while logged in
2. Copy **"API Key (v3 auth)"** вҖ” a 32-character hex string like `554e826839d8ab9d1b65e68d83c61b1d`
3. Do **NOT** use **"API Read Access Token (v4 auth)"** вҖ” that is the long `eyJhbGci...` Bearer token

**If you only have the Bearer token**, extract the v3 key from it:

```python
import base64, json
token = "eyJhbGci...your.bearer.token"
payload = token.split('.')[1]
payload += '=' * (4 - len(payload) % 4)  # fix base64 padding
print(json.loads(base64.b64decode(payload))['aud'])
# prints your v3 API key
```

### вқҢ Browser shows JSON but Python times out

**Cause:** Your browser routes through a system proxy or VPN browser extension that Python's `requests` library does not use. Python makes direct TCP connections and bypasses browser-level proxy settings.

**Fix вҖ” set proxy environment variables before running:**

```powershell
# Find your proxy in browser settings вҶ’ use that address here
$env:HTTPS_PROXY="http://proxy.address:port"
$env:HTTP_PROXY="http://proxy.address:port"
python app.py
```

The DNS change (Solution 1 above) is the cleaner fix as it resolves the root cause.
